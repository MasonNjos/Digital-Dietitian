<h1>Basic Functionality</h1>
The system shall read a cronometer report PDF from input.
The system shall then highlight the nutritents that go above the target by at least 10%.
The system shall display the top 10 foods that contain the high nutrient in decreasing order.
The system shall highlight the nutrients that are under the target by at least 10%.
The system shall recommend 5 foods that are high in the low nutrient in decreasing order.

<h2>Future improvements</h2>
The system shall prompt the user for a weight goal. This goal could be gain weight, lose weight or maintain weight.
The system shall prompt the user for a health goal. This could be control blood glucose, bone health, heart health, etc.
The system shall prompt the user for a fitness goal. This should be different fitness types including weight lifting, calisthenics, running, swimming, gymnastics, sports, etc.
The system shall recommend nutrient goals and foods based on goals and biometric info including weight, height, age, and gender.
The system shall identify trends in biometric data. For example, if a user sees a consistent spike in blood glucose after eating rice, the system should highlight this potential correlation.
The system shall identify trends relating biometric data to exercise data. For example, if a user logged a run and saw lower blood glucose peak for that day, the system should highlight this.
The system shall identify trends correlating exercise to health habits. For example, if a user logs a run and makes better food choices for their goals, the system should highlight this.
